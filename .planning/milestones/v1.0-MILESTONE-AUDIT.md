## Critical Functional Bug (Unchanged from Previous Audit)

### DETC-01–04, RMVL-02: Detection Skipped on Incremental Swaps

**File:** `src/detection/engine.ts` line 115
**Function:** `runDetectionIfNeeded()`

```typescript
// BUG: swaps.timestamp is Unix seconds; wallet.last_checked_at is Unix milliseconds
gt(swaps.timestamp, lastChecked)  // lastChecked ~1000x too large — always false in steady state
```

**Impact:** After a wallet's initial import, no new scam detection fires during monitoring cycles. A wallet that becomes a bundler/sniper/dev wallet after import is never flagged. RMVL-02 (auto-remove on confirmed detection) is non-functional for post-import behavior.

**One-line fix:**
```typescript
const lastCheckedSeconds = Math.floor(wallet.last_checked_at / 1000);
gt(swaps.timestamp, lastCheckedSeconds)
```

**Phase 9 status:** The gap-closure phase created specifically for this fix was never executed. No directory, no plans, no VERIFICATION.md exists for Phase 9.
