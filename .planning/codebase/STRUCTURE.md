## Key File Locations

**Entry Points:**
- `src/cli.ts`: Command-line interface using commander.js, parses --wallet, --file, --days, --export options
- `src/index.ts`: Programmatic entry point exporting scoreWallet(address: string) function
- `tests/test-fetchers.ts`: API fetcher integration test

**Configuration:**
- `.env`: Runtime environment variables (contains API keys - not committed)
- `.env.example`: Template showing required environment variables
- `tsconfig.json`: TypeScript compiler options with path aliases (@types/*, @fetchers/*, etc.)

**Core Logic:**
- `src/fetchers/helius.ts`: HeliusFetcher class with getTransactions() and getTransaction() methods
- `src/fetchers/dexscreener.ts`: DexScreenerFetcher class with getTokenPrice() and getTokenPrices() methods
- `src/types/wallet.ts`: Wallet, WalletAnalysis, WalletMetrics, Position type definitions
- `src/types/transaction.ts`: Transaction, Swap, Helius*, DexScreener* type definitions
- `src/types/config.ts`: Config, ApiConfig, CalculationConfig, CategoryCriteria definitions

**Testing:**
- `tests/test-fetchers.ts`: Tests both Helius and DexScreener fetchers, validates API connections

## Naming Conventions

**Files:**
- Modules: kebab-case (e.g., `helius.ts`, `dexscreener.ts`, `test-fetchers.ts`)
- Barrel exports: lowercase `index.ts`
- Config files: dot-notation (e.g., `.eslintrc.cjs`, `.prettierrc`)
- Test files: suffix `.test.ts` or `.spec.ts` (currently named descriptively like `test-fetchers.ts`)

**Directories:**
- Feature/Layer folders: kebab-case (e.g., `src/calculators`, `src/categorization`, `src/exporters`)
- Group folders by responsibility not by file type
- Config directory at project root: `config/`

**Exports:**
- Barrel files use `export * from './file.js'` to re-export named exports
- `src/types/index.ts` collects all type exports for central import point

## Where to Add New Code

**New Feature (e.g., new metric calculation):**
- Primary code: `src/metrics/` (create new file for category of metrics)
- Tests: `tests/unit/metrics.test.ts` (new test file for feature)
- Types: Add new interface to `src/types/wallet.ts` or create new type file
- Export: Update `src/types/index.ts` if adding new exported types

**New Component/Module (e.g., new exporter format):**
- Implementation: `src/exporters/[format]-exporter.ts`
- Factory function: Export factory function `createXyzExporter()` for consistency with fetcher pattern
- Types: Add export format types to `src/types/export.ts`

**Utilities (e.g., date helper, formatting):**
- Shared helpers: `src/utils/[category].ts` (e.g., `src/utils/dates.ts`)
- Pure functions preferred
- Export via barrel file `src/utils/index.ts`

**Configuration:**
- New config section: `src/types/config.ts` (add interface for new config)
- Config file: `config/[feature].json` (if runtime config files needed)
- Environment variable: `.env` and `.env.example` (for API keys, endpoints)

**Tests:**
- Unit tests: `tests/unit/[module].test.ts`
- Integration tests: `tests/integration/[feature].test.ts`
- Follow existing test pattern in `tests/test-fetchers.ts`
