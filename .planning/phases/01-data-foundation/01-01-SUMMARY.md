## Decisions Made

- **Jest over Vitest:** jest was already in package.json and package was partially configured — no reason to migrate.
- **WAL not in migration SQL:** WAL is a connection-level pragma that must be re-applied each connection open. Putting it in the migration would only run once and be ignored on subsequent connections.
- **pnpm onlyBuiltDependencies:** Required to allow better-sqlite3 native build in pnpm v10's stricter security model. Also needed nvm-installed Linux node (v22.22.1) to compile the native module in WSL — the Windows node.exe cannot run node-gyp.
- **WAL test uses temp file, not :memory::** SQLite in-memory databases always report `journal_mode=memory` regardless of PRAGMA WAL setting. Changed to temp file to accurately verify WAL mode is being set.
- **NODE_OPTIONS=--experimental-vm-modules jest:** Cleaner than `node --experimental-vm-modules jest.js` because it allows pnpm to pass test pattern arguments to jest without them being swallowed by the node invocation.

## Issues Encountered

- WSL + pnpm v10 + Windows node.exe: native module build requires Linux node via nvm. This is an environment constraint that will affect all phases using native modules.
- pnpm `--` argument passthrough to jest: `pnpm test -- --testPathPattern=x` inserts `--` before jest args which jest treats as end-of-flags. Use `pnpm test --testPathPattern=x` (no double-dash) instead.
