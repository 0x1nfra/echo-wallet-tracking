Required sections:

## Prerequisites
- Railway CLI installed (`npm install -g @railway/cli`)
- Logged in: `railway login`
- Git repository linked to Railway project

## Environment Variables
Set these in Railway dashboard (Settings > Variables) or via `railway variables set`:

| Variable | Description | Example |
|---|---|---|
| DATABASE_URL | Absolute path to SQLite file on the mounted volume | `/data/echo.db` |
| PORT | Injected automatically by Railway — do not set manually | (auto) |
| TELEGRAM_BOT_TOKEN | Bot token from @BotFather | `123456:ABC...` |
| HELIUS_API_KEY | Helius RPC API key | `abc123...` |
| SHYFT_API_KEY | Shyft fallback RPC key | `xyz789...` |

Note: DATABASE_URL MUST use the volume mount path (/data by default). A path like `./echo.db` will be lost on every deploy.

## Volume Setup
1. In Railway dashboard, open your service
2. Go to Settings > Volumes
3. Add a volume, mount path: `/data`
4. Redeploy after attaching the volume

## Deploy
```bash
railway up
```
Railway reads railway.toml automatically. The build uses the Dockerfile.

## Verification
After deploy completes:
1. Check health endpoint: `curl https://<your-service>.railway.app/health` — expect `{"status":"ok","uptime":...}`
2. Check Railway logs for startup summary (logged by cli.ts serve command on boot)
3. Confirm DATABASE_URL path exists: check logs for volume-check output

## Troubleshooting

**SQLITE_READONLY or SQLITE_CANTOPEN**
Volume permission issue. Verify the volume is mounted at `/data` and DATABASE_URL points to `/data/echo.db`. Railway volumes mount as root; the service runs as root (Dockerfile does not switch user).

**Health check timeout / deployment stuck**
Host binding issue. Confirm server is listening on `0.0.0.0`, not `127.0.0.1`. Railway's load balancer cannot reach loopback addresses.

**Service exits immediately on startup**
Volume not mounted. The startup sequence does a 30-second retry loop waiting for the volume path. If the path never appears, the service hard-fails. Attach the volume in Railway dashboard and redeploy.
