**1. Add 30m window processing** — prepend to resolveOutcomes() BEFORE the existing 1h/4h/24h loops. Mirror the existing 1h pattern exactly:
```typescript
const due30m = db.select({ id, token_mint, entry_price, tier })
  .from(signal_events)
  .where(and(
    isNull(signal_events.outcome_30m_price),
    lte(signal_events.fired_at, nowMs - 1_800_000),
  ))
  .limit(MAX_PER_CYCLE)
  .all();

for (const event of due30m) {
  const outcomePrice = await fetcher.getTokenPrice(event.token_mint);
  const { status, pct } = classifyOutcome(event.entry_price, outcomePrice, event.tier);
  db.update(signal_events)
    .set({ outcome_30m_price: outcomePrice, outcome_30m_pct: pct, outcome_30m_status: status })
    .where(eq(signal_events.id, event.id))
    .run();
  updatePeakPrice(db, event.id, outcomePrice, nowMs); // see below
  updateMilestones(db, event.id, event.entry_price, outcomePrice, nowMs); // see below
  resolved++;
}
```

**2. Peak price update helper** — extract as a private function `updatePeakPrice(db, eventId, outcomePrice, nowMs)`:
- Fetch current `peak_price` for the event
- If `outcomePrice !== null && (currentPeak === null || outcomePrice > currentPeak)`:
  - Update `peak_price = outcomePrice`, `peak_price_at = nowMs`
- This runs after EVERY window resolution (30m, 1h, 4h, 24h), not just once

**3. Milestone writes helper** — `updateMilestones(db, eventId, entryPrice, outcomePrice, nowMs)`:
- Read milestones from: `const milestones = (process.env.OUTCOME_MILESTONES ?? '50,100,300').split(',').map(Number)` — compute at module load time (not inside the loop)
- For each milestone (50, 100, 300): if `pct * 100 >= milestone` AND the hit flag is not already set:
  - `hit_50=true, hit_50_at=nowMs` (or hit_100, hit_300 for other milestones)
- Use db.select to read current hit flags before deciding — only write if not already set (idempotency)
- Do NOT run if `outcomePrice === null` or `entryPrice === null` or `entryPrice === 0`
