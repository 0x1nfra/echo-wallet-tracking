<objective>
Fix three timestamp unit mismatches that prevent incremental detection and scoring from firing after initial wallet import.

Purpose: The bugs cause runDetectionIfNeeded() and scoreWalletIfNeeded() to always skip — because swaps.timestamp is in Unix seconds (~1_711_000_000) but the comparison values (last_checked_at, calculated_at) are in milliseconds (~1_711_000_000_000). The swaps always appear "older" than last_checked_at, so detection never re-runs, RMVL-02 never fires from new swaps, and the wash-trader window covers ~19 years instead of 7 days.

Output: Three corrected source files and updated/new tests proving the fix. The 237-test suite stays green.
</objective>
