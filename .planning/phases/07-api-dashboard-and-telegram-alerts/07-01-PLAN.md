  <action>
    **Create src/api/cycle-events.ts**:
    ```typescript
    import { EventEmitter } from 'events';

    // Shared singleton: MonitorLoop emits 'cycle' after each signal computation.
    // Fastify SSE route subscribes to this to fan out live updates to browsers.
    export const cycleEmitter = new EventEmitter();
    cycleEmitter.setMaxListeners(50); // allow many concurrent browser connections
    ```

    **Update src/monitor/loop.ts** — add import and emit call:

    1. Add import at the top of loop.ts (after existing imports):
       ```typescript
       import { cycleEmitter } from '../api/cycle-events.js';
       ```

    2. In the `runCycle()` private method (or tick method), after the `computeAllTokenSignals()` call succeeds (inside the try block, after the `console.log('[monitor] signals...')` line), add:
       ```typescript
       cycleEmitter.emit('cycle', { timestamp: Date.now() });
       ```

    The emit must be inside the try block and only fires on successful signal computation. If the signal engine throws, the catch block handles it — cycleEmitter.emit should NOT be in the catch path.

    Ensure the import path uses `.js` extension (ESM convention used throughout the codebase — see existing imports in loop.ts).
  </action>

    3. Run `pnpm test` — all 167 existing tests must still pass. The cycleEmitter emit is a no-op when no listeners are attached, so it should not affect test behavior.
