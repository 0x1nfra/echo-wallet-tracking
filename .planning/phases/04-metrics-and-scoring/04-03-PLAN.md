    **Imports needed:**
    ```typescript

    import { db } from '../db/index.js';
    import { wallets, swaps, wallet_metrics, score_history } from '../db/schema.js';
    import {
      groupIntoClosedTrades,
      calculateWinRate,
      calculateRealizedPnl,
      calculateSharpeRatio,
      calculateMaxDrawdown,
      calculateRecencyScore,
    } from '../metrics/index.js';
    import { composeScore } from './composer.js';
    import type { ComputedMetrics } from './composer.js';
    ```

    **Eligibility gate (SCOR-03):** Applied at the start of scoreWallet — return silently (no error thrown) if:
    - Wallet not found in DB
    - `wallet.history_complete !== true`
    - `wallet.detection_status !== 'confirmed_passing'`
    - `walletSwaps.length < 20`

    **computeMetrics(walletSwaps, nowMs) → ComputedMetrics:**
    - `closedTrades = groupIntoClosedTrades(walletSwaps)`
    - `sharpeRatio = calculateSharpeRatio(closedTrades)` — apply confidence dampener: multiply by `Math.min(1.0, closedTrades.length / 50)`
    - `winRateDecimal = calculateWinRate(closedTrades)`
    - `recencyScore = calculateRecencyScore(walletSwaps, nowMs)`
    - `maxDrawdown = calculateMaxDrawdown(walletSwaps)`
    - `tradeCount = walletSwaps.length`
    - `recentTradeCount`: count walletSwaps with timestamp >= nowMs - 180*24*60*60*1000
    - `distinctTokensTraded`: count distinct token_mints in recent sells (side='sell', timestamp >= cutoff)
    - Return ComputedMetrics object

    **Dormancy guard (per RESEARCH.md recommendations):** After computing metrics, if `recentTradeCount < 1`, update wallets.score to null and return without writing wallet_metrics or score_history.

    **persistScore(walletAddress, metrics, scoreResult, nowMs) → void:**
    - Upsert wallet_metrics: use drizzle's `.insert(...).onConflictDoUpdate(...)` targeting `wallet_address`:
      - Set: win_rate, realized_pnl_sol, sharpe_ratio, max_drawdown, recency_score (raw metric values)
      - Also set: score_total, score_risk_adjusted, score_win_rate, score_consistency_recency, score_activity_health (from scoreResult)
      - Also set: trade_count, recent_trade_count (from metrics), calculated_at = nowMs
    - Update wallets.score: `db.update(wallets).set({ score: scoreResult.total }).where(eq(wallets.address, walletAddress)).run()`
    - Append score_history: `db.insert(score_history).values({ wallet_address: walletAddress, score: scoreResult.total, scored_at: nowMs }).run()`

    **scoreWallet(walletAddress: string, nowMs?: number) → void:**
    ```typescript
    export function scoreWallet(walletAddress: string, nowMs: number = Date.now()): void {
      // 1. Eligibility gate
      // 2. Load all swaps for wallet ordered by timestamp asc
      // 3. Activity floor (< 20 swaps → return)
      // 4. computeMetrics()
      // 5. Dormancy guard (recentTradeCount < 1 → null score, return)
      // 6. composeScore(metrics)
      // 7. persistScore()
    }
    ```

    **scoreWalletIfNeeded(walletAddress: string) → void:**
    - Check wallet_metrics for existing calculated_at; if null, proceed to scoreWallet
    - If calculated_at exists, check for any swaps with timestamp > calculated_at; if none, return
    - Otherwise call scoreWallet(walletAddress)

    **scoreAllEligible() → { scored: number; skipped: number }:**
    - Query wallets where history_complete=true AND detection_status='confirmed_passing'
    - For each, call scoreWallet(); track scored (score written) vs skipped (eligibility failed or dormant)
    - Return counts for CLI display

    NOTE: The `sql` import from drizzle-orm is needed for the onConflictDoUpdate target. Check how existing engine files handle upserts — look at detection engine or schema for patterns. Use drizzle's `conflict_target` parameter targeting the wallet_address unique constraint.
